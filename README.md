# \# Greenhouse Frontend Dashboard 🌿📊

# 

# Інтерфейс системи моніторингу та керування мікрокліматом теплиці. Проєкт розроблений у межах курсу "Проєктування інформаційних систем".

# 

# \## 📝 Опис ідеї (MVP)

# 

# Minimum Viable Product (MVP) — це веб-інтерфейс, який дозволяє користувачу в реальному часі відстежувати стан сенсорів (температура, вологість) та віддалено керувати обладнанням (полив, освітлення).

# 

# \*\*Основні функції:\*\*

# 

# \- \*\*Візуалізація даних:\*\* Відображення показників з датчиків у зручному графічному форматі.

# \- \*\*Керування пристроями:\*\* Можливість ручного перемикання реле (насос, вентилятор).

# \- \*\*Налаштування порогів:\*\* Встановлення критичних значень для автоматичного спрацювання системи.

# \- \*\*Логування:\*\* Перегляд історії змін показників за певний період.

# 

# \## 🛠 Технологічний стек

# 

# \- \*\*Frontend:\*\* JavaScript (Vanilla) / TypeScript

# \- \*\*Стилізація:\*\* CSS3 (Custom properties)

# \- \*\*Інструменти збірки:\*\* Vite / Live Server

# \- \*\*Контроль версій:\*\* Git (GitHub)

# 

# \## 📂 Структура проєкту

# 

# 

# Відповідно до поточної архітектури:

# 

# \- `src/css/` — файли стилів (layout, components, animations).

# \- `src/js/` — клієнтська логіка та робота з API.

# \- `src/static/` — статичні ресурси (іконки, зображення).

# \- `src/types/` — визначення типів (якщо використовується TypeScript).

# \- `src/utils/` — допоміжні функції (форматування дат, обчислення).

# \- `public/` — публічні файли та фавіконки.

# \- `index.html` — головна сторінка застосунку.

# \- `.env` — конфігурація середовища та API-ключі.
